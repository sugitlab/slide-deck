# アウトライン的なメモ

- 今日のテーマはAI時代にプロダクトマネージャーの仕事
- AIによってPMの仕事はどう変わるのかーーーを話すのが鉄板な気がしましたが、ちょっとずらします
- AIをプロダクトに組み込むなら、どんなことを考えておくとよいのかを一緒に考えましょう
- 個人的に、プロダクト開発というプロセスにおけるAI活用がすごく盛り上がっているんだけど、やっぱりプロダクトを良くすることにもっと頭を使いたいなぁと日々思っていて、手段選考みたいな組み立てですが、プロダクトにAIを組み込もうってなったときに、どんな準備をしておくとよいのか、どんな事を考えておくとよいのかについて、考えるきっかけになったらいいなぁと思ってこのテーマにしました。
- ここでのAIはいわゆるLLMのAPIを活用したほげほげを想定してもらって大丈夫ですが、AI=LLMではないのでそのあたりは誤解のないように話したいなと思います
- みなさん、AIは使っていますか？
- では、AIを使ったプロダクト・機能の開発はしていますか？
- 手段先行じゃんと言われるかもですが、ぶちゃけやってみたくないですか？楽しそうだし。
- ってことで、私はやっちゃいましたw
- TechTrain Learning Agent
- 自分で企画して、仕様書いて、実装して、（もちろん各工程でみんなにレビューしてもらって）、リリースしました。これはGitHub Copilotと連携して動く、いわゆるLLMベースです。
- 実はちょっと前、前職のときにも、AIを使ったデータ分析機能（こっちはLLMではなく、DeepLearningを応用した独自開発の手法）みたいなものも、開発してリリースしたことがあります
- そんなこともあり、AIを組み込む場合に考えておかないといけないことってどんな事があったかなーを振り返って整理してみました
- ざっくり三項目
  - 決定論的か、確率論的かの見極め。
  - 確率論的な仕組みがどのレイヤーにあるのか
  - 要件定義への影響
- それぞれ詳細を話していきます
